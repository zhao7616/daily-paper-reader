## 摘要
我们提出了 LoRM（旋转机械语言），这是一个用于多模态旋转机械信号理解和实时状态监测的自监督框架。LoRM 基于这样一种理念：旋转机械信号可以被视为一种机器语言，即局部信号可以被标记化为离散的符号单元，并且可以根据观察到的多传感器上下文预测其未来的演变。与依赖手工设计变换和特征的传统信号处理方法不同，LoRM 将多模态传感器数据重新表述为基于标记的序列预测问题。对于每个数据窗口，观察到的上下文片段以连续形式保留，而每个传感通道的未来目标片段则被量化为离散标记。随后，通过在工业信号上对通用预训练语言模型进行部分微调，实现了高效的知识迁移，避免了从头开始训练大模型的需要。最后，通过跟踪标记预测误差作为健康指标来进行状态监测，其中误差的增加表示性能退化。原位刀具状态监测（TCM）实验证明了稳定的实时跟踪和强大的跨刀具泛化能力，表明 LoRM 为语言建模与工业信号分析之间架起了一座实用的桥梁。源代码可在 https://github.com/Q159753258/LormPHM 公开获取。

## Abstract
We present LoRM (Language of Rotating Machinery), a self-supervised framework for multi-modal rotating-machinery signal understanding and real-time condition monitoring. LoRM is built on the idea that rotating-machinery signals can be viewed as a machine language: local signals can be tokenised into discrete symbolic units, and their future evolution can be predicted from observed multi-sensor context. Unlike conventional signal-processing methods that rely on hand-crafted transforms and features, LoRM reformulates multi-modal sensor data as a token-based sequence-prediction problem. For each data window, the observed context segment is retained in continuous form, while the future target segment of each sensing channel is quantised into a discrete token. Then, efficient knowledge transfer is achieved by partially fine-tuning a general-purpose pre-trained language model on industrial signals, avoiding the need to train a large model from scratch. Finally, condition monitoring is performed by tracking token-prediction errors as a health indicator, where increasing errors indicate degradation. In-situ tool condition monitoring (TCM) experiments demonstrate stable real-time tracking and strong cross-tool generalisation, showing that LoRM provides a practical bridge between language modelling and industrial signal analysis. The source code is publicly available at https://github.com/Q159753258/LormPHM.

#### 1. 核心问题与整体含义
旋转机械（如电机、齿轮箱、机床）在工业中至关重要，其状态监测（Condition Monitoring）直接关系到生产安全与经济效益。传统方法依赖手工设计的特征提取和复杂的信号处理变换，且难以有效融合来自振动、声学、力等不同传感器的多模态数据。
**LoRM 的核心动机**是将旋转机械信号类比为一种“机器语言”：局部信号模式可以被视为“单词”（Token），而机器的动力学演化则构成了“上下文”。论文旨在通过自监督学习，利用预训练语言模型（PLM）的序列建模能力，实现无需人工标注的实时状态监测。

#### 2. 方法论
LoRM 框架将多传感器信号处理重新定义为一个基于 Token 的序列预测问题：
*   **上下文-目标表述（Context-Target Formulation）：** 将每个信号窗口分为两部分：**上下文段**（保留连续波形，提供近期背景）和**目标段**（后续信号）。
*   **信号标记化（Tokenization）：** 对每个传感通道的目标段使用 **K-means 聚类**，将其量化为离散的符号单元（Token）。每个通道拥有独立的码本（Codebook）。
*   **多传感器融合（Code-switching 启发）：** 将不同通道的上下文段切片（Patching）后，按预定义顺序展平为一个统一序列。这模仿了多语言沟通中的“代码切换”，允许 Transformer 主干同时建模时间和跨通道依赖。
*   **模型架构：** 采用预训练语言模型（如 GPT-2）作为主干。通过**部分微调（Partial Fine-tuning）**策略，冻结大部分注意力层，仅训练嵌入层、位置编码和轻量化预测头。
*   **状态监测逻辑：** 训练模型预测未来的目标 Token。在机器健康时，预测误差（交叉熵损失）较低；当机器出现磨损或故障时，信号偏离正常模式，预测误差增大。该误差被定义为**健康指标（HI）**。

#### 6. 主要结论与发现
*   **离散化优势：** 将信号预测转化为离散 Token 分类比直接进行连续波形回归（如 GPT4TS）更稳定，对噪声和振幅波动的敏感度更低。
*   **预训练价值：** 使用在大规模文本或语音上预训练的模型作为初始化，比从头训练模型具有更强的跨工具泛化能力。
*   **架构敏感性：** 自回归预训练模型（如 GPT-2）在跨工具任务中表现优于掩码语言模型（如 BERT），因为其预训练目标与 LoRM 的未来预测目标更一致。
*   **实时监测：** LoRM 能够稳定跟踪刀具磨损趋势，并在达到 ISO 标准阈值时准确触发警报。

#### 7. 优点
*   **创新视角：** 首次系统性地提出了“旋转机械语言”概念，为工业信号处理提供了统一的语言建模视角。
*   **自监督学习：** 无需密集的故障标签，仅需正常运行数据即可训练，极具工业应用潜力。
*   **多模态融合：** 巧妙地通过序列展平解决了异构传感器（力、振动、声音等）的融合难题。
*   **泛化性强：** 实验证明模型在不同刀具之间具有良好的迁移能力。

#### 8. 不足与局限
*   **超参数依赖：** 聚类数量 $K$、窗口长度和 Patch 大小等参数对结果有一定影响，目前仍需经验选择。

*   **工况覆盖：** 实验主要集中在恒定转速的铣削任务，对于转速剧烈波动的非平稳工况（如频繁启停）的适应性尚待进一步验证。
